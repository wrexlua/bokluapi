Guide
Authentication Flow
Every session follows these 5 steps in order. Re-auth is required when a session expires (30 min TTL).
STEP 01
auth
Send Riot JWT + SID → get session_id + envelope
STEP 02
access
Relay Riot's response to the auth envelope → access envelope
STEP 03
heartbeat
Send every ~60 s to keep the session alive
STEP 04
tasks
Poll after each HB for pending ObjectIDs needing report
STEP 05
report
Submit each ObjectID to clear it from the pending list

Each endpoint handles exactly one action — no action field needed.

POST /vgc/session/gateway · Auth
Initiate Session
Opens a new Vanguard session. Send your Riot JWT and hardware SID — receive a session_id and an encrypted envelope to forward to Riot.
POST https://emu.theend.lat/vgc/session/gateway
Request body
FieldTypeDescription
gamestringRequired"valo" or "league"
gametokenstringRequiredRiot JWT (cid:lol)
sidstringRequired (valo)Hardware SID
puuidstringOptionalRiot PUUID
{
  "game":      "valo",
  "gametoken": "eyJhbGciOiJSUzI1NiJ9...",
  "sid":       "SID_hardware",
  "puuid":     "00000000-0000-0000-0000-000000000000"
}
Response — 200
{
  "success":      true,
  "data":         "base64_envelope_for_riot",
  "session_id":   "sess_xxxxxxxxxxxxxxxx",
  "loyalty_uuid": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
}
curl
curl -s -X POST https://emu.theend.lat/vgc/session/gateway \
  -H "Content-Type: application/json" \

POST /vgc/session/heartbeat · Heartbeat
Session Heartbeat
Keeps the Vanguard session alive. Send every ~60 seconds with Riot's latest response. Session expires after 30 minutes of inactivity.
POST https://emu.theend.lat/vgc/session/heartbeat
Request body
FieldTypeDescription
gamestringRequired"valo" or "league"
session_idstringRequiredActive session ID
responsestringRequiredBase64 response received from Riot
{
  "game":       "valo",
  "session_id": "sess_xxxxxxxxxxxxxxxx",
  "response":   "base64_riot_response"
}
Response — 200
{"success": true, "data": "base64_envelope_for_riot"}
POST /vgc/session/tasks · Tasks
Pending Tasks
Returns the list of ObjectIDs that Riot requires you to report. Poll this after each heartbeat or access. An empty array means nothing is pending.
POST https://emu.theend.lat/vgc/session/tasks
Request body
FieldTypeDescription
session_idstringRequiredActive session ID
{
  "session_id": "sess_xxxxxxxxxxxxxxxx"
}
Response — 200
{
  "success": true,
  "tasks": ["objectid_aaa", "objectid_bbb"]
}

tasks is an empty array [] when nothing is pending.

POST /vgc/session/report · Report
Submit Report
Submits a report for a specific ObjectID. Once reported, the ObjectID is removed from the pending tasks list. Repeat for each task returned.
POST https://emu.theend.lat/vgc/session/report
Request body
FieldTypeDescription
gamestringRequired"valo" or "league"
session_idstringRequiredActive session ID
responsestringRequiredBase64 response from Riot
object_idstringRequiredObjectID from the tasks list
{
  "game":       "valo",
  "session_id": "sess_xxxxxxxxxxxxxxxx",
  "response":   "base64_riot_response",
  "object_id":  "objectid_aaa"
}
Response — 200
{"success": true, "data": "base64_envelope_for_riot"}
Reference
HTTP Status Codes
All codes returned by the API and what they mean.
200
OK
Request succeeded. data contains the base64 envelope to forward to Riot.
400
Bad Request
Missing or invalid field. Check the request body against the parameter table.
401
Unauthorized
Invalid or revoked license key, or session has expired — re-auth required.
403
Forbidden
Tier basic is only authorized for action=auth. Upgrade to full for all actions.
404
Not Found
Route does not exist.
405
Method Not Allowed
Use POST for gateway endpoints, GET for /healthz.
429
Too Many Requests
Rate limit hit. Slow down requests — especially Riot gateway calls (280 s guard).
500
Internal Server Error
Server-side failure. Retry after a short delay.
